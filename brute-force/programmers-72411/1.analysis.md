# 메뉴 리뉴얼

## 📌 문제 요약

각 손님의 주문에서 가능한 모든 조합을 생성하고, 모든 주문에서 가장 많이 주문된 조합을 찾는 문제입니다.

---

## 🔍 문제 설명

### 핵심 규칙

1. **조합 생성**: 각 손님의 주문에서 가능한 모든 조합을 생성
2. **빈도 카운트**: 생성된 조합이 몇 명의 손님에게서 나왔는지 카운트
3. **최대 빈도 선택**: 각 코스 크기별로 가장 많이 주문된 조합 선택
4. **최소 조건**: 최소 2명 이상의 손님에게서 주문된 조합만 유효

### 예시 설명

**입력:**
```
orders = ["ABCFG", "AC", "CDE", "ACDE", "BCFG", "ACDEH"]
course = [2, 3, 4]
```

**각 조합의 빈도:**
- "AC": 4번 (1번, 2번, 4번, 6번 손님)
- "CDE": 3번 (3번, 4번, 6번 손님)
- "ACDE": 2번 (4번, 6번 손님)
- "BCFG": 2번 (1번, 5번 손님)

**출력:**
```
result = ["AC", "CDE", "ACDE", "BCFG"]
```

---

## 📥 입력 조건

- orders 배열의 크기: 2 ≤ N ≤ 20
- orders 배열의 각 원소는 크기가 2 이상 10 이하인 문자열
- 각 문자열은 알파벳 대문자로만 이루어져 있음
- 각 문자열에는 같은 알파벳이 중복해서 들어있지 않음
- course 배열의 크기: 1 ≤ C ≤ 10
- course 배열의 각 원소는 2 이상 10 이하인 자연수가 오름차순으로 정렬됨

---

## 📤 출력 조건

- 각 코스요리 메뉴의 구성을 문자열 형태로 배열에 담아 반환
- 사전 순으로 오름차순 정렬
- 각 문자열도 알파벳 오름차순으로 정렬
- 가장 많이 함께 주문된 메뉴 구성이 여러 개라면 모두 포함
