# 구간 합 (Prefix Sum)

구간 합은 배열의 누적 합을 미리 계산해 두고, 임의의 구간 합을 O(1)에 구하는 기법입니다.
여러 번의 구간 합 질의가 들어오는 문제에서 매우 자주 사용됩니다.

---

## 📚 핵심 개념

배열 `A[1..N]`에 대해 누적 합 배열 `P`를 다음처럼 정의합니다.

- `P[0] = 0`
- `P[i] = P[i-1] + A[i]` (1 ≤ i ≤ N)

그러면 구간 `[l, r]`의 합은:

- `sum(l, r) = P[r] - P[l-1]`

으로 즉시 계산할 수 있습니다.

---

## 🔧 기본 알고리즘

### 1) 누적 합 배열 만들기

```java
long[] prefix = new long[n + 1];
for (int i = 1; i <= n; i++) {
    prefix[i] = prefix[i - 1] + arr[i];
}
```

### 2) 구간 합 질의 처리

```java
long rangeSum = prefix[r] - prefix[l - 1];
```

---

## ⏱️ 복잡도

- 누적 합 배열 전처리: O(N)
- 각 구간 합 질의: O(1)
- Q개의 질의 전체: O(N + Q)
- 추가 공간: O(N)

---

## 💡 언제 쓰는가

- 정적 배열(값 변경이 없음)에서 구간 합 질의가 많을 때
- 2차원 격자의 부분 합(2D Prefix Sum)을 빠르게 구할 때
- 구간 합을 활용한 조건 탐색(합이 K 이상/이하인 구간 찾기)의 기반 자료로 사용할 때

---

## ⚠️ 주의사항

- 인덱스를 1-based로 맞추면 `P[r] - P[l-1]` 형태가 깔끔합니다.
- 값의 범위가 크면 `int` 대신 `long`을 사용해야 오버플로우를 피할 수 있습니다.
- 배열 값이 자주 변경되면 Prefix Sum 단독으로는 비효율적이며, 세그먼트 트리/펜윅 트리를 고려해야 합니다.
