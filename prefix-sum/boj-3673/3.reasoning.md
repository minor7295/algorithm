# Main.java 설계 근거 정리

이 문서는 BOJ 3673(나눌 수 있는 부분 수열)을 풀 때,
왜 "누적합 나머지 + 빈도 카운팅"으로 접근해야 하는지 정리합니다.

## 1. 완전 탐색 가능성 판단

연속 부분 수열은 시작점/끝점 조합으로 총 `N(N+1)/2`개입니다.

- `N = 50,000`일 때 구간 개수는 약 12.5억
- 각 구간 합을 직접 계산하면 시간 제한(1초) 내 처리가 불가능

따라서 O(N^2) 계열 접근은 제외해야 합니다.

## 2. 핵심 수학 근거

누적합을 `prefix[k] = A1 + ... + Ak`라고 할 때,
구간 `[i+1, j]`의 합은 `prefix[j] - prefix[i]`입니다.

이 구간 합이 `d`로 나누어떨어지는 조건은:

- `(prefix[j] - prefix[i]) % d == 0`
- 즉 `prefix[j] % d == prefix[i] % d` 입니다.

그러므로 **두 누적합의 나머지가 같으면 그 사이 구간은 조건을 만족**합니다.

## 3. 카운팅 방식 선택

각 나머지 `r`(0 ~ d-1)에 대해,
누적합에서 `r`이 나온 횟수를 `cnt[r]`라고 두면
같은 나머지 2개를 고르는 경우의 수 (조합의 개수) = `cnt[r] * (cnt[r] - 1) / 2` 이고,
- 전체 정답 = 모든 `r`에 대한 위 값의 합

여기서 `prefix[0] = 0`도 반드시 포함해야 하므로,
초기값으로 `cnt[0] = 1`을 먼저 넣고 시작합니다.

## 4. 복잡도 및 자료형

- 시간 복잡도: O(N) (테스트케이스당 한 번 순회)
- 공간 복잡도: O(d) (`cnt` 저장)
- 정답 최대치는 `N(N+1)/2`까지 가능하므로 `answer`는 `long` 사용
