# 알고리즘: 판단 내용을 실제 코드로 옮긴 구성


## 1. 스택 구조를 배열로 구현

스택에는 "아직 최대 구간이 확정되지 않은 막대의 인덱스"를 저장합니다.
현재 코드는 `Deque`가 아니라 고정 배열 스택을 사용합니다.

- `int[] stack = new int[n + 1]`
- `int top = -1`

이 구성으로 push/pop을 상수 시간에 처리합니다.

```java
int[] stack = new int[n + 1];
int top = -1;
```

## 2. 높이 감소 시 pop하며 구간 확정

현재 위치 `i`에서 `cur = h[i]`를 보고,
`h[stack[top]] > cur`인 동안 pop을 반복합니다.

pop된 인덱스를 `mid`라고 할 때 확정되는 값은 다음과 같습니다.

- `height = h[mid]`
- 오른쪽 경계: `right = i - 1`
- 왼쪽 경계:
- 스택이 비면 `left = 0`
- 스택이 남아 있으면 `left = stack[top] + 1`
- `width = right - left + 1`

그리고 정사각형 변 길이 후보를
`side = min(height, width)`로 계산해 `answer`를 갱신합니다.

```java
while (top >= 0 && h[stack[top]] > cur) {
    int mid = stack[top--];
    long height = h[mid];

    int left = (top < 0) ? 0 : stack[top] + 1;
    int right = i - 1;
    long width = right - left + 1L;

    long side = Math.min(height, width);
    if (side > answer) {
        answer = side;
    }
}
stack[++top] = i;
```

## 3. 같은 높이와 마지막 정리

비교 조건을 `>`로 두었기 때문에 같은 높이는 즉시 pop되지 않고 스택에 유지됩니다.
이 후보들은 더 낮은 높이가 나올 때 한꺼번에 정리됩니다.

또한 `h[n] = 0` 센티넬을 추가하고 루프를 `i = 0 .. n`까지 돌려,
끝까지 남아 있는 막대들도 빠짐없이 pop/계산되도록 처리합니다.

## 4. 입력 처리와 자료형 선택

`Main.java`는 `BufferedReader + StringTokenizer` 기반 `next()`로 입력을 읽습니다.

- `N`은 `int`로 처리합니다.
- 높이 배열 `h`와 정답 `answer`는 `long`으로 처리합니다.

높이 자체는 `10^9`까지이고 정사각형 변 길이의 최댓값은 `N`에 의해 제한되지만,
계산 일관성과 안전성을 위해 `long`을 사용한 구성입니다.

```java
int n = Integer.parseInt(next());
long[] h = new long[n + 1];
for (int i = 0; i < n; i++) {
    h[i] = Long.parseLong(next());
}
h[n] = 0; // Sentinel to flush remaining bars.
```

## 5. 복잡도

각 인덱스는 스택에 최대 1번 push, 최대 1번 pop되므로
전체 시간복잡도는 `O(N)`입니다.

높이 배열과 인덱스 스택을 사용하므로 공간복잡도는 `O(N)`입니다.
문제 제한 `N <= 300,000`에서 충분히 통과 가능한 구성입니다.
