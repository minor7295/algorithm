# 회의실 배정

## 📌 문제 요약

한 개의 회의실이 있는데 이를 사용하고자 하는 N개의 회의에 대하여 회의실 사용표를 만들려고 한다. 각 회의 I에 대해 시작시간과 끝나는 시간이 주어져 있고, 각 회의가 겹치지 않게 하면서 회의실을 사용할 수 있는 회의의 최대 개수를 찾아보자.

---

## 🔍 문제 설명

* N개의 회의가 있고, 각 회의는 시작 시간과 끝나는 시간이 주어진다.
* 회의는 한번 시작하면 중간에 중단될 수 없다.
* 한 회의가 끝나는 것과 동시에 다음 회의가 시작될 수 있다.
* 회의의 시작시간과 끝나는 시간이 같을 수도 있다. (시작하자마자 끝나는 것으로 생각)
* 목표: 겹치지 않게 회의실을 사용할 수 있는 회의의 최대 개수

---

## 📥 입력 조건

* 첫째 줄에 회의의 수 N(1 ≤ N ≤ 100,000)이 주어진다.
* 둘째 줄부터 N+1 줄까지 각 회의의 정보가 주어진다.
  * 공백을 사이에 두고 회의의 시작시간과 끝나는 시간이 주어진다.
  * 시작 시간과 끝나는 시간은 2^31-1보다 작거나 같은 자연수 또는 0이다.

---

## 📤 출력 조건

* 첫째 줄에 최대 사용할 수 있는 회의의 최대 개수를 출력한다.

---

## 💡 문제 핵심 해석

이 문제는 **활동 선택 문제(Activity Selection Problem)**의 변형으로, **그리디 알고리즘(Greedy Algorithm)**을 사용하여 해결할 수 있다.

### 핵심 조건

* **회의는 중단 불가**: 한번 시작하면 끝까지 진행
* **동시 시작 가능**: 한 회의가 끝나는 것과 동시에 다음 회의가 시작될 수 있음
* **목표**: 겹치지 않게 최대 개수의 회의 선택

### 그리디 접근 가능성

**끝나는 시간이 빠른 회의를 우선 선택**하는 그리디 전략이 최적해를 보장한다:

* 끝나는 시간이 빠른 회의를 선택하면
* 남은 시간이 최대한 많아져서
* 더 많은 회의를 선택할 수 있음

**예시**:
```
회의: [(1,4), (3,5), (0,6), (5,7), (3,8), (5,9), (6,10), (8,11), (8,12), (2,13), (12,14)]

끝나는 시간 기준 정렬:
[(1,4), (3,5), (0,6), (5,7), (3,8), (5,9), (6,10), (8,11), (8,12), (2,13), (12,14)]

그리디 선택:
1. (1,4) 선택 → 마지막 끝 시간 = 4
2. (3,5) 스킵 (시작 시간 3 < 4)
3. (0,6) 스킵 (시작 시간 0 < 4)
4. (5,7) 선택 (시작 시간 5 >= 4) → 마지막 끝 시간 = 7
5. (3,8) 스킵 (시작 시간 3 < 7)
6. (5,9) 스킵 (시작 시간 5 < 7)
7. (6,10) 스킵 (시작 시간 6 < 7)
8. (8,11) 선택 (시작 시간 8 >= 7) → 마지막 끝 시간 = 11
9. (8,12) 스킵 (시작 시간 8 < 11)
10. (2,13) 스킵 (시작 시간 2 < 11)
11. (12,14) 선택 (시작 시간 12 >= 11) → 마지막 끝 시간 = 14

결과: 4개 [(1,4), (5,7), (8,11), (12,14)]
```

➡️ **끝나는 시간이 빠른 회의를 우선 선택**하는 것이 최적해이다.

---

## 🧠 해결 전략 개요

1. **끝나는 시간 기준으로 정렬** (끝나는 시간이 같으면 시작 시간 기준 정렬)
2. **첫 번째 회의 선택**
3. **이후 회의들을 순회하면서**:
   * 현재 회의의 시작 시간이 마지막 선택한 회의의 끝 시간 이상이면
   * 해당 회의를 선택하고 마지막 끝 시간 업데이트
4. **선택한 회의의 개수 반환**

## ✨ 예시

### 예시 1: 문제에서 제공한 예제

**입력**
```
11
1 4
3 5
0 6
5 7
3 8
5 9
6 10
8 11
8 12
2 13
12 14
```

**처리 과정**
```
회의 목록:
[(1,4), (3,5), (0,6), (5,7), (3,8), (5,9), (6,10), (8,11), (8,12), (2,13), (12,14)]

1단계: 끝나는 시간 기준 정렬
[(1,4), (3,5), (0,6), (5,7), (3,8), (5,9), (6,10), (8,11), (8,12), (2,13), (12,14)]

2단계: 그리디 선택
- (1,4) 선택 → count = 1, lastEnd = 4
- (3,5) 스킵 (3 < 4)
- (0,6) 스킵 (0 < 4)
- (5,7) 선택 (5 >= 4) → count = 2, lastEnd = 7
- (3,8) 스킵 (3 < 7)
- (5,9) 스킵 (5 < 7)
- (6,10) 스킵 (6 < 7)
- (8,11) 선택 (8 >= 7) → count = 3, lastEnd = 11
- (8,12) 스킵 (8 < 11)
- (2,13) 스킵 (2 < 11)
- (12,14) 선택 (12 >= 11) → count = 4, lastEnd = 14

결과: 4개
```

**출력**
```
4
```

---

## 📝 정리

이 문제는 **그리디 알고리즘**을 사용하여 해결할 수 있는 전형적인 활동 선택 문제이다.

* **핵심 전략**: 끝나는 시간이 빠른 회의를 우선 선택
* **시간 복잡도**: O(N log N) - 정렬 + O(N) - 순회 = O(N log N)
* **공간 복잡도**: O(N) - 회의 배열 저장
