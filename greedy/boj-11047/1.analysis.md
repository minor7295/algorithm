# 동전 0

## 📌 문제 요약

준규가 가지고 있는 동전은 총 N종류이고, 각각의 동전을 매우 많이 가지고 있다.

동전을 적절히 사용해서 그 가치의 합을 K로 만들려고 한다. 이때 필요한 동전 개수의 최솟값을 구하는 프로그램을 작성하시오.

---

## 🔍 문제 설명

* N종류의 동전이 있고, 각 동전을 무한히 많이 사용할 수 있다.
* 동전의 가치의 합을 정확히 K로 만들어야 한다.
* 사용하는 동전의 개수를 최소화해야 한다.
* 동전의 가치는 오름차순으로 주어지고, 큰 동전은 작은 동전의 배수이다.

---

## 📥 입력 조건

* 첫째 줄에 N과 K가 주어진다. (1 ≤ N ≤ 10, 1 ≤ K ≤ 100,000,000)
* 둘째 줄부터 N개의 줄에 동전의 가치 Ai가 오름차순으로 주어진다.
  * (1 ≤ Ai ≤ 1,000,000, A1 = 1, i ≥ 2인 경우에 Ai는 Ai-1의 배수)

---

## 📤 출력 조건

* 첫째 줄에 K원을 만드는데 필요한 동전 개수의 최솟값을 출력한다.

---

## 💡 문제 핵심 해석

이 문제는 **그리디 알고리즘(Greedy Algorithm)**을 사용하여 해결할 수 있는 전형적인 동전 문제이다.

### 핵심 조건

* **동전의 가치는 오름차순으로 주어짐**: A1 < A2 < ... < AN
* **큰 동전은 작은 동전의 배수**: Ai는 Ai-1의 배수 (i ≥ 2)
* **각 동전을 무한히 많이 사용 가능**: 제한 없이 사용 가능

### 그리디 접근 가능성

큰 동전이 작은 동전의 배수라는 조건 때문에 **그리디 알고리즘이 최적해를 보장**한다:

* 큰 동전을 최대한 많이 사용하면
* 남은 금액을 작은 동전으로 채울 수 있고
* 이것이 최소 개수를 보장함

예를 들어, 동전이 [1, 5, 10, 50, 100, 500, ...]이고 K = 4200이라면:

```
4200 = 500 × 8 + 100 × 2
     = 8 + 2 = 10개 (잘못된 계산 예시)

실제로는:
4200 = 5000 × 0 + 1000 × 4 + 500 × 0 + 100 × 2
     = 0 + 4 + 0 + 2 = 6개
```

➡️ **큰 동전부터 최대한 많이 사용**하는 것이 최적해이다.

---

## 🧠 해결 전략 개요

1. **동전을 내림차순으로 정렬** (또는 역순으로 순회)
2. **큰 동전부터 최대한 많이 사용**
   * 현재 동전으로 만들 수 있는 최대 개수 = K / Ai
   * 사용한 동전 개수 추가
   * 남은 금액 = K % Ai
3. **남은 금액이 0이 될 때까지 반복**

---

## ✨ 예시

### 예시 1: 문제에서 제공한 예제

**입력**
```
10 4200
1
5
10
50
100
500
1000
5000
10000
50000
```

**처리 과정**
```
K = 4200
동전: [50000, 10000, 5000, 1000, 500, 100, 50, 10, 5, 1]

50000: 4200 / 50000 = 0개, 남은 금액 = 4200
10000: 4200 / 10000 = 0개, 남은 금액 = 4200
5000:  4200 / 5000  = 0개, 남은 금액 = 4200
1000:  4200 / 1000  = 4개, 남은 금액 = 200
500:   200 / 500    = 0개, 남은 금액 = 200
100:   200 / 100    = 2개, 남은 금액 = 0

총 동전 개수: 4 + 2 = 6개
```

**출력**
```
6
```

```
10 4790
```

**처리 과정**

```
10000: 4790 / 10000 = 0개, 남은 금액 = 4790

```

## 📝 정리

이 문제는 **그리디 알고리즘**을 사용하여 해결할 수 있는 전형적인 동전 문제이다.

* **핵심 조건**: 큰 동전이 작은 동전의 배수 → 그리디 최적해 보장
* **전략**: 큰 동전부터 최대한 많이 사용
* **시간 복잡도**: O(N) - 동전 종류 수만큼 반복
* **공간 복잡도**: O(N) - 동전 배열 저장
