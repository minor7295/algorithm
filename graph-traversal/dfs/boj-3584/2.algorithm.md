# 알고리즘: 판단 내용을 실제 코드로 옮긴 구성

## 1. 부모 정보 기반 접근의 반영

3번 파일의 핵심 판단("트리에서 부모 포인터를 따라 올라가면 조상 체인을 얻을 수 있다")은
코드에서 `parent[]` 배열 구성으로 반영된다. 입력 `A B`를 읽을 때 `parent[B] = A`로 저장하면,
각 노드에서 루트 방향으로 상향 이동이 가능해진다.

```java
for (int i = 0; i < n - 1; i++) {
    int a = nextInt();
    int b = nextInt();
    parent[b] = a;
    hasParent[b] = true;
}
```

## 2. 깊이 맞추기 판단의 반영

두 노드의 깊이가 다르면 더 깊은 쪽을 먼저 올려야 한다는 판단은
`while (depth[u] > depth[v]) u = parent[u];` 형태로 옮긴다.

이 단계가 끝나면 `u`, `v`가 같은 레벨에 있으므로,
이후에는 동시에 올리기만 하면 된다.

```java
while (depth[u] > depth[v]) {
    u = parent[u];
}
while (depth[v] > depth[u]) {
    v = parent[v];
}
```

## 3. 동시 상승으로 LCA 확정하는 판단의 반영

깊이가 같아진 두 노드를 함께 올리다가 처음 같아지는 노드가 LCA라는 판단은
아래 루프로 직접 구현된다.

```java
while (u != v) {
    u = parent[u];
    v = parent[v];
}
// u(또는 v)가 LCA
```

추가 자료구조 없이도 부모 포인터만으로 정답 노드를 얻는다.

## 4. 깊이 계산과 루트 판별 반영

깊이 맞추기 로직을 쓰려면 각 노드 깊이가 필요하므로,
부모가 없는 유일 노드를 루트로 찾은 뒤 DFS/BFS 1회로 `depth[]`를 채운다.

```java
int root = 1;
for (int i = 1; i <= n; i++) {
    if (!hasParent[i]) {
        root = i;
        break;
    }
}

buildDepth(root, parent, depth, children);
```
