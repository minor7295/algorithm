# 베스트 앨범

## 📌 문제 요약

스트리밍 사이트에서 장르 별로 가장 많이 재생된 노래를 두 개씩 모아 베스트 앨범을 출시하려 합니다. 노래는 고유 번호로 구분하며, 노래를 수록하는 기준은 다음과 같습니다.

1. 속한 노래가 많이 재생된 장르를 먼저 수록합니다.
2. 장르 내에서 많이 재생된 노래를 먼저 수록합니다.
3. 장르 내에서 재생 횟수가 같은 노래 중에서는 고유 번호가 낮은 노래를 먼저 수록합니다.

**목표**: 베스트 앨범에 들어갈 노래의 고유 번호를 순서대로 반환

---

## 🔍 문제 설명

### 핵심 규칙

1. **장르 우선순위**: 장르별 총 재생 횟수가 많은 순서대로 수록
2. **노래 우선순위**: 같은 장르 내에서 재생 횟수가 많은 순서대로 수록
3. **동점 처리**: 재생 횟수가 같으면 고유 번호가 낮은 노래를 먼저 수록
4. **수록 개수**: 각 장르에서 최대 2개씩만 수록 (장르에 속한 곡이 하나라면 하나만 선택)

---

## 📥 입력 조건

* `genres`: 노래의 장르를 나타내는 문자열 배열
  * `genres[i]`는 고유번호가 i인 노래의 장르
  * 길이: 1 이상 10,000 이하
* `plays`: 노래별 재생 횟수를 나타내는 정수 배열
  * `plays[i]`는 고유번호가 i인 노래가 재생된 횟수
  * 길이: `genres`와 같음
* 장르 종류: 100개 미만
* 장르에 속한 곡이 하나라면, 하나의 곡만 선택
* 모든 장르는 재생된 횟수가 다름

---

## 📤 출력 조건

* 베스트 앨범에 들어갈 노래의 고유 번호를 순서대로 담은 배열을 반환
* 반환 타입: `int[]`

---

## 💡 문제 핵심 해석

이 문제는 **해시맵(HashMap)과 정렬(Sorting)**을 활용하여 해결할 수 있는 문제입니다.

**핵심 접근:**
1. 해시맵으로 장르별 노래 정보 그룹화
2. 장르별 총 재생 횟수 계산
3. 장르를 총 재생 횟수 순으로 정렬
4. 각 장르 내 노래를 재생 횟수 순으로 정렬 (같으면 고유 번호 오름차순)
5. 각 장르에서 최대 2개씩 선택

---

## ✨ 예시

### 예시 1

**입력**
```
genres = ["classic", "pop", "classic", "classic", "pop"]
plays = [500, 600, 150, 800, 2500]
```

**출력**
```
[4, 1, 3, 0]
```

**설명**: 
* 장르별 재생 횟수:
  - pop: 600 + 2,500 = 3,100회
  - classic: 500 + 150 + 800 = 1,450회
* 장르 우선순위: pop → classic
* pop 장르: 4번(2,500회), 1번(600회) → [4, 1]
* classic 장르: 3번(800회), 0번(500회) → [3, 0]
* 최종: [4, 1, 3, 0]

---

## 📝 정리

이 문제는 **해시맵을 사용한 그룹화와 정렬**을 통해 해결할 수 있습니다.

시간 복잡도 O(N log N)로 해결할 수 있으며, N ≤ 10,000이므로 충분히 효율적입니다.
