# 세 수의 합

## 📌 문제 요약

N개의 자연수로 이루어진 집합 U가 있다. 이 중에서 적당히 세 수를 골랐을 때, 그 세 수의 합 d도 U 안에 포함되는 경우가 있을 수 있다. 이러한 경우들 중에서, 가장 큰 d를 찾으라.

예를 들어 {2, 3, 5, 10, 18}와 같은 집합이 있다고 하자. 2+3+5 = 10이 되고, 이 수는 집합에 포함된다. 하지만 3+5+10 = 18이 되고, 이 경우가 세 수의 합이 가장 커지는 경우이다.

---

## 🔍 문제 설명

* N개의 자연수로 이루어진 집합 U가 주어진다
* 세 수 a, b, c를 선택하여 a + b + c = d를 만족하는 d를 찾는다
* d는 반드시 집합 U에 포함되어야 한다
* x, y, z, k가 서로 같아도 된다 (같은 수를 여러 번 사용 가능)
* 가능한 d 중에서 가장 큰 값을 찾아야 한다

---

## 📥 입력 조건

* 첫째 줄에 자연수 N (5 ≤ N ≤ 1,000)이 주어진다
* 다음 N개의 줄에 차례로 U의 원소가 하나씩 주어진다
* 주어진 U는 집합이 되므로 입력되는 두 수가 같아서는 안 된다
* U의 원소는 200,000,000보다 작거나 같은 자연수이다
* 답이 항상 존재하는 경우만 입력으로 주어진다

---

## 📤 출력 조건

* k번째 수가 최대가 되도록 하는 것이 목적이다
* k번째 수를 출력하면 된다

---

### 📌 핵심 제약 조건

1. **세 수의 합이 집합에 포함되어야 함**
   - a + b + c = d이고, d는 집합 U에 포함

2. **같은 수를 여러 번 사용 가능**
   - x, y, z, k가 서로 같아도 됨
   - 예: 2 + 2 + 2 = 6 (모두 같은 수 사용)

3. **가장 큰 d를 찾아야 함**
   - 가능한 모든 d 중에서 최대값

**예시 분석:**
```
집합 U: {2, 3, 5, 10, 18}

가능한 조합:
- 2 + 3 + 5 = 10 ✓ (10은 집합에 포함)
- 3 + 5 + 10 = 18 ✓ (18은 집합에 포함)
- 2 + 2 + 2 = 6 ✗ (6은 집합에 없음)
- 3 + 3 + 3 = 9 ✗ (9는 집합에 없음)

최적 해: 18 (3 + 5 + 10 = 18)
```


---

## ✨ 예시

### 예시 1

**입력**
```
5
2
3
5
10
18
```

**출력**
```
18
```

**설명**: 
* 3 + 5 + 10 = 18
* 18은 집합에 포함되며, 가능한 값 중 가장 큼

### 예시 2

**입력**
```
6
1
2
3
4
5
6
```

**출력**
```
6
```

**설명**: 
* 1 + 2 + 3 = 6
* 6은 집합에 포함되며, 가능한 값 중 가장 큼

### 예시 3

**입력**
```
5
1
2
3
4
10
```

**출력**
```
10
```

**설명**: 
* 1 + 2 + 7 = 10이지만 7은 집합에 없음
* 2 + 3 + 5 = 10이지만 5는 집합에 없음
* 1 + 4 + 5 = 10이지만 5는 집합에 없음
* 3 + 3 + 4 = 10 ✓ (3은 집합에 포함)
* 또는 2 + 4 + 4 = 10 ✓
* 10은 집합에 포함되며, 가능한 값 중 가장 큼

---

## 📝 정리

이 문제는 **세 수의 합이 집합에 존재하는지 확인하는 문제**로, 두 수의 합을 미리 계산하고 해시셋을 활용하면 효율적으로 해결할 수 있다.

시간 복잡도 O(N²) 또는 O(N³)로 해결할 수 있으며, N ≤ 1,000이므로 충분히 효율적이다.
