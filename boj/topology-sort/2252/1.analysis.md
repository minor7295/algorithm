# 줄 세우기 (위상 정렬)

## 📌 문제 요약

N명의 학생을 키 순서대로 줄 세우려 한다. 모든 학생의 키를 직접 비교할 수 없기 때문에, 일부 학생 쌍에 대한 **상대적인 키 비교 결과**만이 주어진다.

비교 결과를 모두 만족하도록 학생들을 앞에서부터 순서대로 배치하는 프로그램을 작성한다.

---

## 🔍 문제 설명

* 학생은 총 **N명**이며 번호는 `1 ~ N`
* 일부 학생 쌍에 대해 키 비교 결과가 주어진다
* 비교 정보 `A B`는 **학생 A가 학생 B보다 앞에 서야 한다**는 의미
* 모든 비교 조건을 만족하는 줄 세우기 결과를 출력해야 한다

---

## 📥 입력 조건

* 첫째 줄에 정수 `N`, `M`

  * `1 ≤ N ≤ 32,000`
  * `1 ≤ M ≤ 100,000`
* 다음 `M`개의 줄에 걸쳐 두 정수 `A B`가 주어짐

  * 이는 **A → B** 관계를 의미

---

## 📤 출력 조건

* 학생들을 **앞에서부터 순서대로 한 줄에 출력**
* 가능한 답이 여러 개일 경우 **아무거나 하나 출력 가능**

---

## 💡 문제 핵심 해석

* 학생을 **정점(Vertex)**, 비교 결과를 **방향 간선(Edge)** 으로 하는 **방향 그래프**로 모델링 가능
* `A가 B보다 앞에 서야 한다` → `A → B`
* 모든 조건을 만족하는 순서를 찾는 문제이므로

➡️ **사이클이 없는 방향 그래프(DAG)의 위상 정렬 문제**

> **참고**: 이 문제에서는 입력이 항상 유효한 DAG라고 가정합니다. 
> 실제 구현에서는 사이클 검출이 필요할 수 있지만, 본 문제에서는 사이클이 존재하지 않습니다.

---

## 🧠 해결 전략 개요

* 진입 차수(In-degree)를 활용한 **Kahn 알고리즘** 또는
* DFS 기반 위상 정렬 사용 가능
* 입력 크기가 크므로 `O(N + M)` 시간 복잡도 필요

---

## ✨ 예시

### 예시 1: 기본 케이스

**입력**
```
3 2
1 3
2 3
```

**출력** (가능한 예)
```
1 2 3
```

**설명**: 1번과 2번은 순서가 정해지지 않았으므로 둘 다 가능. 3번은 반드시 마지막.

---

### 예시 2: 완전한 순서

**입력**
```
4 3
1 2
2 3
3 4
```

**출력**
```
1 2 3 4
```

**설명**: 완전한 순서가 주어진 경우, 위상 정렬 결과는 유일함.

---

### 예시 3: 독립적인 학생들

**입력**
```
3 0
```

**출력** (가능한 예)
```
1 2 3
```

**설명**: 비교 정보가 없는 경우, 어떤 순서든 가능.

---

### 예시 4: 여러 경로가 있는 경우

**입력**
```
5 4
1 3
2 3
3 4
3 5
```

**출력** (가능한 예)
```
1 2 3 4 5
```

**설명**: 1번과 2번은 순서 무관, 3번은 1,2 이후, 4번과 5번은 3번 이후이지만 서로 순서 무관.

---

## 📝 정리

이 문제는 **부분 순서(partial order)** 가 주어졌을 때 이를 만족하는 **전체 순서(total order)** 를 구하는 전형적인 위상 정렬 문제이다.
정답이 여러 개일 수 있으므로, 조건만 만족하면 어떤 순서든 허용된다.
